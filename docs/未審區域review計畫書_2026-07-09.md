## 0. 給 Opus 4.8 的執行守則（沿用上一輪，硬性）

1. **開工前**：`git fetch origin` + `git status` 確認乾淨且與 origin/main 同步；逐條開工前先重讀該 finding 行號段落確認現況相符（行號以 `1512ee5` 為基準會漂移；不符 → 記錄後跳過回報，不硬修）。
2. **只修本文件列出的項目**；「選配」由執行者裁量。
3. **每批**：`python -m pytest -q` 全綠（基準見開工當下）才進下一批；每個 fix 加回歸測試。
4. **發佈紀律**：修完 → `python .claude/codex.py review --uncommitted` → 認真處理正確性/安全/blocking → 過了才 `python scripts/push_helper.py "<訊息>"`（自動 bump version + 重生 manifest；**絕不手動 git commit+push src**）。
5. **⚠ OneDrive 教訓**（上一輪實際發生兩次靜默還原）：每批修完當日內 push；高風險大檔（`main.py`/`uvb_dose.py`）每個小項改完**立即 commit 到分支**，並 `git diff > %TEMP%\<批次>.patch` 備份；push 後 `git show HEAD:<檔>` 驗證 commit 內容。
6. **⚠ pytest settings 污染陷阱**（上一輪踩到）：pytest 下 `get_settings_dir()` 解析到 `site-packages/pytest/settings/`。任何寫真實 settings 的測試都要隔離到 `tmp`（見背景任務 task_e3f4e471）。
7. 語意慣例：唯讀 fail-open、寫入/資料落地 fail-closed；Tk 只在主緒碰；**醫療計算/寫回（UVB 劑量、打卡）＝不確定就不自動動作，留給醫師**，絕不猜危險值。
8. Windows：檔案 IO 一律 `encoding='utf-8'`；subprocess 對外部指令比照既有 `errors="replace"` 慣例；PowerShell 輸出在繁中機是 cp950（見上一輪 AC-06 教訓，需要中文就強制 UTF-8）。
9. 安全紅線：repo 維持 **public**；`settings/` 與含 PII（員編/GSM/姓名/帳密）檔絕不 commit。
