```python
class RosterService:
    def __init__(self, storage: RosterStorage): ...

    # ── 讀取組裝 ─────────────────────────────────────────────
    def build_context(self, scope: str, ym: str) -> SolveContext:
        """讀 config/ledger/holiday_duty/week_colors/month 檔 → SolveContext。
        - members: config r_members/vs_members → Member.from_dict
        - holidays: storage.holidays_set()
        - leaves/must_duty: month["leaves"][scope] / month["must_duty"][scope]
          （JSON ISO 字串 → date；scope 鍵 "r"/"vs"）
        - annual_holiday: load_holiday_duty()[scope]
        - locks: month[f"{scope}_duty"] 中 locked==True 的格 → {date: person}
        - ledger: load_ledger()[scope]
        - prev_last_weekend: storage.prev_month_last_weekend(ym, scope)
        - 回傳前 .prepare()"""

    # ── 求解與落地 ───────────────────────────────────────────
    def run_solve(self, scope, ym, allow_disable_color=False) -> SolveResult:
        """build_context → solve_duty。不落地（讓 UI 先預覽）。"""

    def render_report(self, scope, ym, result) -> str:
        """重建 ctx → build_report(ctx, result, label)。UI「報告」鈕在 accept 前
        （result 在手但未落地）用它預覽；accept 後可直接讀 month["report_"+scope]。"""

    def accept_solution(self, scope, ym, result) -> None:
        """使用者接受排班結果 → 一次完成（順序固定）：
        （※ 已實裝：不收 ctx，內部由 build_context 重建 → 免 UI 夾帶過期 ctx）
        0. status != "ok" → ValueError；月檔已 finalized → FinalizedMonthError（早擋，
           免帳本已 settle 卻寫不了月檔的半套狀態）
        1. month[f"{scope}_duty"] = {iso: {"person","locked","source":"auto"}}
           ※ 已鎖定格保留原 person/locked/source 不覆蓋
        2. month["last_weekend"][scope] = result.last_weekend
        3. month["report_"+scope] = render_report(...)
        4. ledger: settle_month(ledger, scope, ym, result.points_by_person)（內含回滾）
        5. storage.save_ledger + storage.save_month（月檔後存：ledger 先壞仍可重跑）"""

    # ── 手動編輯（每次都立即存檔+審計）──────────────────────────
    def set_cell(self, scope, ym, d: date, person: str|None, via="manual"): ...
        """改格（None=清空）。source="manual"。append audit {ts,cell,old,new,via}。
        改完跑 quick_validate 回警告清單（不阻止,設計文件 §16.4）。"""
    def toggle_lock(self, scope, ym, d: date) -> bool: ...
    def set_leaves(self, scope, ym, member_id, dates: set[date]): ...
    def set_must(self, scope, ym, member_id, dates: set[date]): ...
    def finalize(self, ym, on: bool): ...   # month["finalized"]=on（解除需 force save）

    def quick_validate(self, scope, ym) -> list[Precheck]:
        """以目前月檔內容跑 run_prechecks + 「週末成對是否被手動改破」檢查，
        給 UI 顯示黃色警告。不求解。"""
```

def extract_week_colors(pdf_path: str, year: int) -> tuple[dict, list]:
    """回 ({week_key: "pink"/"green"}, warnings)。
    1. page.get_text("words") → 日期數字 word 及 bbox

    4. 未覆蓋的週 → 不寫（UI 顯示灰=未設定）"""

### 7.4 solve_day.py（PGY/Clerk 填充器；純函式+決定性）
```python
@dataclass
class DayFillContext:
    d: date; session: str
    rooms: list[str]                 # 該時段開診房(升冪)
    pgy_avail: list[str]; clerk_avail: list[str]   # 已扣請假
    counters: FairCounters           # 見下
    biopsy_open: bool
    room_capacity: int = 2

class FillStep:  # 一步驟一類別,依序執行,各步驟回報人話 log 行
    def run(self, ctx, slots, log): ...
PIPELINE = [治療室Step, 切片室Step, ClerkSeedStep, PgyMixStep,
            ClerkOverflowStep, RestStep]           # 設計文件 §3.6 五步驟+放假
```
- `FairCounters`（存月檔 day_slots 統計重算即可，不另落地）：
  `tx_total[pgy]`(治療室次數)、`tx_wed_pm[pgy]`、`rest[pid]`(放假)、`biopsy_done[clerk]`(梯次內)。
- **輪選 key（決定性鐵律）**：`(count, last_date_assigned or date.min, 代號字典序)` 取最小。
- 治療室Step：pgy_avail 空 → slots 缺+log 紅字（不硬塞）。週三下午用 `tx_wed_pm` counter 優先、同分再看 `tx_total`。
- 切片室Step：只在 biopsy_open；候選=本梯 `biopsy_done==False` 者優先，全 done 則正常輪。
- month_solve_day(ym)：逐 (工作日×早/午) 跑 pipeline → day_slots + 決策 log + 警告（切片輪不到名單=梯末 done==False 者）。
- 測試重點：五步驟順序、1C+1P 混搭、無 Clerk 月（115/07 情境：`101 AD/102 B` 型輸出）、放假輪平均、週三下午僅治療室、切片開放不足警告、請假、決定性。
