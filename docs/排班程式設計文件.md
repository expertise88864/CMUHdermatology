## 1. 目標概述

以 Tkinter 桌面程式（沿用現有 `scheduler.py` 骨架與 cmuh_common 基建）實作皮膚科四種排班：

| 分頁 | 內容 | 演算法 |
|---|---|---|
| R 排班 | 住院醫師值班（一線），整月、週一~週日 | OR-Tools CP-SAT |
| VS 排班 | 主治醫師值班（三線/二線），整月、週一~週日 | OR-Tools CP-SAT（同 solver 換約束集） |
| PGY 排班 | 週一~五早/午跟診＋照光＋治療室 | 確定性貪婪填充（無 solver） |
| Clerk 排班 | 週一~五早/午跟診＋切片室，兩週一梯次 | 確定性貪婪填充 |
| 設定 | 人員/參數/年度假日指定/格網編輯 | — |

另有「決策報告」視圖（每次產生班表後的完整思考過程，見 §11）。

**核心架構原則**（使用者明確要求「隨時可能更改排班邏輯、要有足夠擴充性」）：
1. 排班邏輯 100% 純函式、與 UI 完全分離、全部可 pytest。
2. 規則採「規則註冊表」：每條規則獨立宣告（id/hard-soft/人話描述），新增規則＝加一個檔案＋一個測試，不動 solver 主體。
3. 所有數字參數（點數、容量、房間順序…）進 config，不寫死。
4. 所有讀寫走單一 `storage.py`（未來跨機同步只換後端）。
5. 固定 random seed：同輸入必同輸出（重排可重現）。

## 2. 人員與名詞

- **R（住院醫師，一線）**: R1/R2/R3 各最多一位，未來可能更少（名單可增刪；固定值班週幾綁「人」不綁級職）。
- **VS（主治醫師，三線）**: 目前 6 位，代號 D吳伯元 / J張廖年峰 / R陳駿升 / S沈冠宇 / L謝佳陵 / T蔡李澄。清單可增刪。
- **PGY**: 代號 A/B/C…，預設 6 人，實際常 2-3 人，**每月人數手動調**。
- **Clerk**: 代號 1/2/3…，預設 8 人，實際常 4-6 人，兩週一梯次輪替，**梯次起始必為週一**、不綁月份（跨月實體）。有些月份沒有 Clerk（如 115/07）。
- **時段**: 早診(上午)/午診(下午)/晚診(晚上)。PGY/Clerk 只排早+午。
- **診間**: 101/102/103…（依當時段實際開診）；**照光**（PGY）、**治療室**（PGY）、**切片室**（Clerk）為獨立房間。
- **點數**: 平日值班 1 點、假日(週六日) 2 點、國定假日 1 點。

### 3.6 PGY/Clerk 混排規則（每時段填充**七步驟**，已定案；實作見 solve_day.py）
```
輸入: 該時段開診診間清單(依房號升冪)、可用 PGY、可用 Clerk、診間容量上限(預設2)
1. 照光      ← 1 位 PGY（含週三下午；最優先。公平輪選：總照光次數最少者優先，週三下午另計）
2. 治療室    ← 1 位 PGY（週三下午休診不排。公平輪選同上，各自獨立計次）
3. 切片室(若該時段有開) ← 1 位 Clerk（優先選「本梯次尚未輪過切片室」者）
4. ClerkSeed ：每個開診診間先各放 1 位 Clerk（101→102→103 依房號序；Clerk 依代號輪轉）
5. PgyMix    ：每診第 2 位優先放 PGY（形成 1 Clerk + 1 PGY 混搭）
6. ClerkOverflow：還有 Clerk 沒工作 → 補進剩餘容量
7. Rest      ：還有人沒位子 → 放假（放假次數輪平均）
※ 該月無 Clerk（如 115/07）→ 步驟 3/4/6 跳過，診間直接由 PGY 填（範例：101 AD/102 B）
```

### 3.7 開診格網（PGY/Clerk 的前提資料）— ⚠️ 含重要修正
- 資料來源：主程式 `settings/cache_master_schedule.json`（`{醫師:{weekday:[{session,is_self_paid}]}}` 週模板）。
- `is_self_paid=true`（自費/美容診）**不算診間、不排 PGY/Clerk**。
- 休診（該月特定日停診）→ 格網顯示「休診」且不排人。**實作**：停診由日排班分頁「本月停診…」（`_ClinicClosureDialog`→`service.set_clinic_closed`）寫入月檔 `grid_overrides[iso][session].closed_rooms`；`clinic_grid.month_grid` 據此把該室排除，自動排班不再排入。停診時一併清掉既有班表中該室未鎖定的指派（RS-03/05：清後清空 day_report、撞鎖定回報、寫 audit）。
- **修正**：實際診間**非連續編號填充**。範例（115-07 門診輪值表）：週一早診開 101+103（102 空）。所以格網模型是「**週模板：weekday×session → 開放診間號清單**」（在設定頁維護一張「門診輪值表」編輯器，可從 cache_master_schedule 的醫師數推導初值，但以手動維護為準），再展開成月度逐日格網＋逐日覆寫（休診/加診）。
- 學生填充順序=開放診間**房號升冪**（101→102→103，跳過未開的）。
- 排 8 月班時醫院網掛可能未開放 8 月 → 一律以週模板展開＋手動覆寫，不依賴即時網掛。

### 3.8 行事曆色塊（實作：決定性演算法，非 PDF 抽取）
- **原方案（PDF 抽取，已作廢）**：曾設計用 PyMuPDF `get_drawings()` 從 `1_115年行事曆.pdf` 抽色塊矩形。實作階段改為**不依賴 PDF**——避免換年度/換版面即失效、也免帶個資 PDF。
- **實作（`calendar_colors.py`）**：以**決定性演算法**產生週色——粉紅／淺綠**每 4 週交替**（依 ISO 週序），全程式可重現、換年度自動延續。UI 設定頁提供**手動覆蓋**任一週色（存 `week_colors.json`）。排班只讀週色表。
- 規則語意不變：某人連續兩個「週末」值班 ⇔ 兩週末所屬週色塊必須**不同**；同色→禁止連週。

## 6. Solver 設計（R/VS 共用，CP-SAT）

- 變數：`duty[day] ∈ 人員集合`（one-hot BoolVar 矩陣）。
- 硬約束：每日恰一人；請假日排除；指定日/年度假日表固定；固定週幾（R 專用，該人未請假時該日=該人）；假日成對（週六=週日；三連休段整段相等，「連休段」定義=與週末相鄰的國定假日串成的連續假日區塊）；色塊連週（對每對相鄰週末 w、w+1：若週色相同，則 ∀p: weekend_p[w] + weekend_p[w+1] ≤ 1；w=本月第一週末時，上月最後週末值班人由上月檔讀入為常數）；鎖定格=常數。
- 軟目標（加權和最小化，權重進 config）：
  1. `|每人點數 −(目標+帳本結轉調整)|` 總和（主目標）
  2. 9-11 班範圍偏離罰分
  3. （可擴充目標掛在規則註冊表）
- **放寬階梯**（無解時逐級重試，報告標紅每級動作）：
  - L0 全規則 → L1 放寬 9-11 範圍（內建自動）→ L2 放寬次要軟公平 → L3 停用色塊連週（**必須跳窗經使用者確認**才套用）。
  - 永不放寬：請假、指定值班、假日成對、每日一人。
  - 仍無解 → 用 CP-SAT assumptions 找出最小衝突規則集，人話報告（例：「8/22-23 週末無人可排：R1 請假、R2 被色塊擋、R3 已被指定 8/29 且同色」）。
- 求解 determinism：`random_seed` 固定、`num_search_workers=1`。
- 月結：解出後計算每人實際點數 − 目標 → 寫入 ledger（月份重排時先回滾該月已入帳分錄再重記）。

## 7. PGY/Clerk 填充器設計

`solve_day.py`：對月內每個(日,時段)依 §3.6 **七步驟**填充（照光→治療室→切片→ClerkSeed→PgyMix→ClerkOverflow→Rest）。每一步驟為一個填充步驟，順序可調、參數注入；照光最優先且含週三下午，治療室週三下午休診。公平輪選一律用「(次數, 上次輪到距今, 代號)」排序，決定性可重現。每一步輸出人話 log 進報告。

## 8. 規則註冊表框架

```python
@register_rule(scope="R", kind="hard", rule_id="weekend_pair")
class WeekendPair(Rule):
    描述 = "週六+週日(含相連國定假日)須同一人"
    def precheck(self, ctx) -> list[Warning]: ...   # 排前人話檢查
    def apply(self, model, ctx): ...                # 對 CP-SAT 下約束
    # soft 規則另有 def objective_terms(self, model, ctx)
```
- solver 從註冊表按 scope 組裝；停用/放寬=跳過該規則重解。
- 報告自動列出「本次啟用規則＋參數」。
- **新增/修改排班邏輯的 SOP**：加 rule class → 加測試 → （必要時）設定頁曝光參數 → 更新本文件 §3。

## 9. 決策報告（report.py）

四段式，同步寫入 月份檔、automation_ui.log、UI 報告視圖：
```
[輸入] 人員/請假/指定/帳本結轉/週色表/開診格網摘要
[預檢] 每條規則的人話檢查結果（✓/⚠/✗），衝突在排班前就列出
[過程] R/VS：放寬階梯層級、每格指派理由（固定/指定/平衡）
       PGY/Clerk：逐時段五步驟填充 log
[結算] 每人 平日班/假日班/點數/帳本異動；治療室/切片室/放假 次數表
[警告] 未排入清單＋原因、放寬動作、切片室輪不到名單、治療室空缺
```
> **實作差異（2026-07 回寫）**：R/VS 決策報告為完整四段。PGY/Clerk 日排班報告目前只產出 `[過程]`（逐時段填充 log）與 `[警告]`，**無獨立 `[結算]` 次數表**——即時結算改由日排班分頁的快速檢查（`quick_validate_day`，RS-07）與匯出的 PGY-Clerk 工作表承擔。若日後需要日排班 `[結算]` 段，列為待辦。

## 16. 補遺（v1.1 — 二次盤查找到的缺口，實作時一併吸收）

1. **國定假日清單的資料源**：點數計算（平日假日=1點）、診間自動休診、PGY/Clerk 不排、三連休判定，都需要「哪些日期是國定假日」。定案：`holiday_duty.json` 的日期鍵**本身就是假日清單**（每個國定假日都有指定值班，兩者必然同集合）；由設定頁的年度假日指定表**手動維護**（原「從行事曆 PDF 文字抽取初始種子」隨 §3.8 PDF 方案一併作廢）。平日的國定假日：診間全休、照光/治療室**不開**、PGY/Clerk 不排（該日不在開診格網內，日填充器不會走到）。
2. **clinic_template 增加欄位**：每診加選填「醫師名」＋每週幾加選填「晨會」列——匯出 Word 門診輪值表（早/午/晚×101-103×週一~六，格值=醫師名+R跟診）需要，排班演算法本身不用。
3. **config 人員名冊補欄位**：職稱/GSM簡碼/備註（選填）——匯出 Word「病房、急診值班醫師分配表」表格需要。
4. **手動改格的規則檢查策略**：手動編輯**允許違反任何規則**（使用者最大），但即時跑 precheck 並在格上/警告面板顯示（例：改完週六不同於週日→黃色警告「假日未成對」），**不阻止儲存**。
5. **ortools 版本釘選**：CP-SAT 不同版本可能給出不同（但皆合法）解，破壞「同輸入同輸出」。deps 安裝時釘選版本（如 `ortools==9.x.y`），升版視為有意識的決策。
6. **請假輸入 UX**：實務單人單月可達 12 天請假（115/07 範例），輸入介面用「月曆點選切換」多日期選取器，不用逐日打字。
7. **民國年轉換**：匯出檔標題/內文用民國年（115年07月），寫一個共用 util（民國↔西元）。
8. **R 人數退化**：R 只剩 1-2 位時不 crash——1 位＝全月同一人（點數平衡無意義、自動略過），2 位照常求解；9-11 範圍自動放寬（L1 已涵蓋）。
9. **開發環境注意（重要）**：本 repo 位於 Desktop（疑似 OneDrive 同步範圍），2026-06-25 曾發生**編輯中的原始碼被雲端舊版靜默還原**導致改動遺失。大量開發前：暫停 OneDrive 同步、或將 repo 移出同步資料夾、或每完成一段立即 commit（進了 git 就安全）。
10. **治療室/切片室的「人話」空缺報告**：治療室無 PGY 可排、切片室該梯開放數不足時，報告需列到「人＋日期」粒度（已在 §9/§3.5，此處強調驗收必測）。
